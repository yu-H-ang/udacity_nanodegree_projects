Implementation: Outlier Detection¶

Detecting outliers in the data is extremely important in the data preprocessing step of any analysis. The presence of outliers can often skew results which take into consideration these data points. There are many "rules of thumb" for what constitutes an outlier in a dataset. Here, we will use Tukey's Method for identfying outliers: An outlier step is calculated as 1.5 times the interquartile range (IQR). A data point with a feature that is beyond an outlier step outside of the IQR for that feature is considered abnormal.

In the code block below, you will need to implement the following:

  • Assign the value of the 25th percentile for the given feature to Q1. Use np.percentile for this.
  • Assign the value of the 75th percentile for the given feature to Q3. Again, use np.percentile.
  • Assign the calculation of an outlier step for the given feature to step.
  • Optionally remove data points from the dataset by adding indices to the outliers list.

NOTE: If you choose to remove any outliers, ensure that the sample data does not contain any of these points!
Once you have performed this implementation, the dataset will be stored in the variable good_data.

Additionally, if a signifiant amount of variance is explained by only two or three dimensions, the reduced data can be visualized afterwards.

Question 6¶

What are the advantages to using a K-Means clustering algorithm? What are the advantages to using a Gaussian Mixture Model clustering algorithm? Given your observations about the wholesale customer data so far, which of the two algorithms will you use and why?

Answer: K-Means clustering has hard assignment, which means each data point only belongs to one cluster. It is a simple algorithm and it is very fast. But under some circumstances it can fall in local minima and results in wrong clusters. So usually we use the best result of many runs to avoid this situation. GMM is a soft clustering algorithm. It assumes that all the data are generated from a Gaussian mixture distribution, and it uses a Expectation-Maximisation process to find the paratmeters of the distribution. It is slower than K-means. Some of the k-means' drawbacks make me want to choose GMM in this project. To get a good custering result, k-means assumes evenly sized clusters, or at least nearly equal size, and same vaiance for different variables. But both of them are not necessarily true for our customer data. Since GMM can handle these situations So I choose the GMM algorithm to do the clustering.

The wholesale distributor is considering changing its delivery service from currently 5 days a week to 3 days a week. However, the distributor will only make this change in delivery service for customers that react positively. How can the wholesale distributor use the customer segments to determine which customers, if any, would reach positively to the change in delivery service?
Hint: Can we assume the change affects all customers equally? How can we determine which group of customers it affects the most?

Answer: In this case, A/B testing is used as a method of comparing two versions the delivery service against each other to determine which one performs better. We have divided the customers into 3 different segments with different spending habits. Of course we cannot simply assume the change of delivery service affect all customers equally. To determine how this change affect different groups of customers, we can run 3 A/B tests on each of the customer segments concurrently, with the experimental group being the new 3-days-a-week service and control group the current 5-days-a-week service. In each A/B test we randomly assign one of two variants to each customer then see if the customers have an overall positive or negative react in each of the 3 clusters. Then we will get the information about how the change of delivery service affect each type of segments. The wholesale distributer can apply different strategies on different customer segments to achieve the optimal performance.
